# Session 2026-07-05 — T-016 Reports + Dashboard

## What was done
- Models: ReportDefinition (code, query_key → builders registry,
  default_params, `scheduled` cron storage-only, is_active) + ReportRun
  (format pdf/xlsx/csv, params, status lifecycle, file, error, timings).
  Both audited. 5 definitions seeded by data migration.
- `builders.py`: registry of 5 dataset builders returning ReportData
  (title/columns/rows/summary), optional date_from/date_to params:
  enrollment_stats, verification_outcomes, case_activity, duplicates
  (DEDUP candidates → potential duplicate identities, capped 500),
  clearance_issuance (incl. fees collected).
- `renderers.py`: csv (utf-8-sig BOM), xlsx (openpyxl, bold header,
  31-char sheet-title limit), pdf (platypus table, striped rows).
- Execution: start_report_run → Celery run_report (eager-aware);
  queued→running→done|failed w/ error retained; file saved under
  reports/<code>/.
- Endpoints: definitions read-only, POST /reports/run/ → 202 run payload,
  runs list/detail, download (400 until done; **audited as EXPORT**),
  GET /dashboard/kpis/. ReportPermission sup/aud/admin (run = read-only
  export, so all three may POST); KpiPermission all staff roles.
- KPIs: ROLE_BLOCKS map → per-role block sets (admin/sup: 7 blocks;
  operator: enrollments/applications/certificates; investigator:
  enrollments/matching/alerts; auditor: audit/verification). Blocks:
  enrollments today/week, applications pending/submitted_today, matching
  running_jobs/jobs_today/hit_rate (None when nothing decided),
  certificates issued today/total, alerts open, audit events_today,
  verification today.
