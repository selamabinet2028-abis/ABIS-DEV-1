## What was done
- `preprocessing.crypto`: Fernet encrypt/decrypt via `ABIS_FIELD_KEY`
  (dev default in settings + .env.example; prod.py REQUIRES env — fail fast;
  clear ImproperlyConfigured on bad key). lru_cached Fernet instance.
- `preprocessing.services`: `quality_score` (contrast + Laplacian variance →
  NFIQ-like 1–5; noise ≥4, flat gray = 1), `extract_template` (GRID16 —
  16×16 equalized grayscale, deterministic, prefix `GRID16:`),
  `build_nist_meta` (ANSI/NIST-ITL-flavored: dims, format, dpi, sha256),
  `sha256_hex`. Real SDK replaces function bodies, signatures stay (ADR-016).
- `enrollment.models`: Enrollment (person PROTECT, station PROTECT nullable,
  operator SET_NULL, status, completed_at), BiometricRecord (denormalized
  person FK, modality finger|palm|face, NIST positions validated per
  modality, image file, sha256, quality_score, accepted, nist_meta JSONB,
  captured_by), BiometricTemplate (OneToOne record, engine, version,
  encrypted template_bytes BinaryField).
- `enrollment.services`: `capture_biometric` (score → threshold
  `ABIS_QUALITY_THRESHOLD` (2) → record persisted either way; accepted →
  encrypted template), `complete_enrollment` (requires ≥1 accepted; sets
  completed; `trigger_dedup` stub returns None until T-008).
- Endpoints: enrollments CRUD (no hard delete; operator auto-set),
  `/biometrics/` multipart action (ext whitelist + size cap + decodability),
  `/complete/`, biometric-records read-only + `/image/` FileResponse with

  FK needs it; T-012 extends (booking, TimeSlot).
