# Session 2026-07-04 — T-005 Audit (insert-only AuditLog)

## What was done
- `audit.AuditLog` per DATABASE_DESIGN: actor FK nullable + actor_username
  snapshot, action (create/update/delete/search/view/export/system_error),
  entity ("app.Model"), entity_id, entity_repr, changes JSONB, ip, user_agent,
  `at`; index (entity, entity_id); ordering -at.
- Immutability (application-level REVOKE — ADR-014): instance save() on
  existing row raises, instance delete() raises, queryset update()/delete()
  raise (`AuditImmutabilityError`); bulk_update goes through update() → raises.
- `AuditContextMiddleware` (after AuthenticationMiddleware): stores request in
  a contextvar; `services._request_meta()` derives actor/ip (XFF-aware)/UA.
- `signals.py`: generic pre_save (stash persisted values) / post_save (create
  row with full new values, update row with old→new diff) / post_delete
  (snapshot) receivers, connected in AuditConfig.ready() for every label in
  `ABIS_AUDITED_MODELS` (currently accounts.User, accounts.Role,
  basedata.OrgUnit). Masking: `ABIS_AUDIT_MASK_FIELDS` → `***` (password);
  noise: `ABIS_AUDIT_IGNORE_FIELDS` (last_login, updated_at) skipped; empty
  diffs write nothing; `raw` (fixture) saves skipped; FKs recorded as
  `<field>_id`; JSON-safe conversion for UUID/datetime/Decimal.
- `services.write_audit / audit_instance / log_search` — the only write path;
  log_search ready for person/biometric endpoints (T-006+).
- `/api/v1/audit-logs/` ReadOnlyModelViewSet, `IsAdmin | IsAuditorReadOnly`,
  FilterSet: entity, entity_id, actor (username snapshot), action,
  date_from/date_to (ISO, gte/lte on `at`); search + ordering.
- Shared test fixtures promoted to `backend/conftest.py` (api_client,
  make_user, auth_client) — usable by every app's tests.
